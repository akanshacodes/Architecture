# 5️⃣ WHAT IS A CLIENT?

A client is any software or device that sends requests to a server.

---

# 📌 Examples

| Client | Type |
|---|---|
| Chrome | Browser |
| Firefox | Browser |
| Edge | Browser |
| Mobile App | Client Application |

---

# 📌 Responsibilities of a Client

- Send requests
- Receive responses
- Display information to users

---

# 🔥 Example

When a user opens:

```text
google.com
```

the browser sends a request to Google’s server.

---

# 6️⃣ WHAT IS A SERVER?

A server is a machine or software that:
- Receives requests
- Processes requests
- Sends responses back to clients

---

# 📌 Responsibilities of a Server

- Store website files
- Process requests
- Manage traffic
- Serve webpages
- Run applications

---

# 🌐 Basic Communication

```text
Client → Request
Server → Response
```

# 9️⃣ HOW A WEBSITE WORKS

---

# 🌐 Website Working Flow

---

## STEP 1

User opens browser.

---

## STEP 2

User enters URL.

Example:

```text
https://netflix.com
```

---

## STEP 3

Browser sends request to server.

---

## STEP 4

Web server receives request.

---

## STEP 5

Server processes request.

---

## STEP 6

Server sends response.

---

## STEP 7

Browser renders webpage.

---

# 🔥 Complete Flow

```text
Browser
   ↓ Request
Web Server
   ↓ Response
Browser
   ↓
Website Visible
```

---

# 🔟 BROWSER REQUEST LIFECYCLE

This is one of the most important concepts in web architecture.

---

# 🌐 Lifecycle

```text
1. User enters URL
2. DNS resolves domain
3. Browser contacts server
4. Server processes request
5. Response is generated
6. Browser renders webpage
```

---

# 📌 Example

```text
youtube.com
```

The browser first finds the server IP address and then communicates with the server.
