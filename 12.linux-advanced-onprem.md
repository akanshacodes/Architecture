# 2. Linux System Information Commands

These commands help monitor and inspect system details.

| Command | Purpose |
|---|---|
| `lscpu` | Shows CPU information |
| `free -h` | Displays RAM usage |
| `df -h` | Shows disk storage usage |
| `cat /etc/os-release` | Displays operating system details |
| `hostname -I` | Shows IP address |
| `uptime` | Shows how long system has been running |

---

## 2.1 CPU Information

```bash
lscpu
```

Displays:
- CPU architecture
- Number of cores
- Threads
- Processor details

---

## 2.2 Memory Information

```bash
free -h
```

### Purpose
Displays:
- Total RAM
- Used RAM
- Free RAM

### `-h`
Human-readable format (MB/GB).

---

## 2.3 Disk Usage Information

```bash
df -h
```

### Purpose
Displays:
- Total disk size
- Used storage
- Available storage

---

## 2.4 Operating System Information

```bash
cat /etc/os-release
```

### Purpose
Displays:
- OS Name
- Version
- Linux distribution

---

## 2.5 IP Address

```bash
hostname -I
```

### Purpose
Shows system IP address.

---

## 2.6 System Uptime

```bash
uptime
```

### Purpose
Shows:
- How long the system has been running
- Current load average

# 4. Process Management

Processes are programs running in the system.

| Command | Purpose |
|---|---|
| `ps` | Shows running processes |
| `top` | Live process monitoring |
| `kill` | Terminates a process |

---

## 4.1 Process Status

```bash
ps
```

### Purpose
Displays currently running processes.

---

## 4.2 Real-Time Process Monitoring

```bash
top
```

### Purpose
Linux version of Task Manager.

Displays:
- CPU usage
- RAM usage
- Running processes

### Exit Top
Press:
```bash
q
```

---

## 4.3 Kill a Process

```bash
kill PID
```

Example:
```bash
kill 1234
```

### Purpose
Stops a running process using Process ID (PID).

# 6. Package Management in Linux

APT is a package manager used in Ubuntu/Debian systems.

---

## 6.1 List Packages

```bash
apt list
```

### Purpose
Displays available software packages.

---

## 6.2 Update Package Information

```bash
apt update
```

### Purpose
Refreshes latest package information from repositories.

---

## 6.3 Install Software

```bash
apt install nginx
```

### Purpose
Installs NGINX web server.

---

## 6.4 List Installed Packages

```bash
apt list --installed
```

### Purpose
Shows all installed software.

---

# 7. Output Redirection

Linux allows saving command outputs into files.

---

## 7.1 Overwrite (`>`)

```bash
apt list --installed > installed.txt
```

### Purpose
- Removes old content
- Writes new content

### Flow

```text
Command Output → File
```

---

## 7.2 Append (`>>`)

```bash
command >> file.txt
```

### Purpose
Adds new content below existing content.

### Flow

```text
Old Content
New Content Added Below
```

# 8. Grep Command

`grep` is used for searching text patterns.

---

## Syntax

```bash
grep "keyword" filename
```

Example:
```bash
grep "nginx" installed.txt
```

### Purpose
Searches for:
- Words
- Text
- Patterns

inside files.

---

# 9. Pipe (`|`) Operator

Pipe transfers output of one command as input to another command.

---

## Syntax

```bash
command1 | command2
```

---

## Example

```bash
apt list --installed | grep "nginx"
```

---

## Working Flow

```text
apt list --installed
        ↓
List of installed packages
        ↓
grep "nginx"
        ↓
Only nginx-related results
```

# 11. Systemctl Commands

`systemctl` manages Linux services.

| Command | Purpose |
|---|---|
| `systemctl start nginx` | Starts NGINX |
| `systemctl stop nginx` | Stops NGINX |
| `systemctl restart nginx` | Restarts NGINX |
| `systemctl status nginx` | Checks NGINX status |

---

## Start NGINX

```bash
systemctl start nginx
```

---

## Stop NGINX

```bash
systemctl stop nginx
```

---

## Restart NGINX

```bash
systemctl restart nginx
```

---

## Check Status

```bash
systemctl status nginx
```
