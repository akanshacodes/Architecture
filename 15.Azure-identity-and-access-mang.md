# 4. Manual vs Automated Infrastructure Provisioning

Infrastructure resources can be provisioned manually or through automation.

---

# 4.1 Manual Provisioning

Infrastructure is created using graphical portals or manual commands.

### Examples

- Creating VMs through Azure Portal
- Manually configuring networks
- Installing applications manually

---

## Advantages

- Better conceptual understanding
- Easier for beginners
- Useful for troubleshooting

---

## Limitations

- Time-consuming
- Error-prone
- Difficult to scale

---

# 4.2 Automated Provisioning

Infrastructure is provisioned using scripts or Infrastructure as Code (IaC) tools.

---

## Advantages

- Consistency
- Speed
- Repeatability
- Scalability
- Reduced human error

---

## Important Learning Principle

A strong understanding of manual provisioning is essential before implementing automation.

---

# 5. Imperative vs Declarative Infrastructure Management

Infrastructure automation approaches are categorized into two models.

---

# 5.1 Imperative Approach

Engineers define step-by-step instructions.

---

## Example Workflow

```text
1. Create Virtual Machine
2. Create Network
3. Attach Storage
4. Configure Security Rules
```

---

## Characteristics

- Procedural
- Sequential execution
- Full operational control

---

# 5.2 Declarative Approach

Engineers define the desired end state, and the automation tool determines how to achieve it.

---

## Example

```text
Deploy a Linux VM with 4 GB RAM and public IP access.
```

---

## Characteristics

- Desired state configuration
- Simplified infrastructure management
- Higher automation efficiency

---

# 12. Authentication & Authorization

Authentication and Authorization are the foundational pillars of cloud security.

---

# 12.1 Authentication

Authentication verifies user identity.

---

## Core Question

```text
Who are you?
```

---

## Common Authentication Methods

- Username/password
- Multi-Factor Authentication (MFA)
- Biometrics
- OAuth
- Single Sign-On (SSO)

---

# 12.2 Authorization

Authorization determines what actions an authenticated user can perform.

---

## Core Question

```text
What are you allowed to do?
```

---

## Examples

| User | Permission |
|---|---|
| Reader | View resources only |
| Contributor | Modify resources |
| Owner | Full administrative access |

---

# 13. Token-Based Authentication

Modern cloud platforms primarily use token-based authentication mechanisms.

---

# Authentication Flow

```text
User Login
     ↓
Identity Verification
     ↓
Token Generation
     ↓
Authorized Resource Access
```

---

## What is a Token?

A token is a temporary digital credential used to validate user identity and permissions.

---

## Advantages

- Stateless authentication
- Enhanced security
- Faster validation
- Reduced credential exposure

# 16. Role-Based Access Control (RBAC)

RBAC is a security model where permissions are assigned based on roles rather than individual users.

---

## Why RBAC is Important

- Centralized access management
- Principle of least privilege
- Improved security
- Simplified administration

---

## Common Azure Roles

| Role | Permissions |
|---|---|
| Reader | Read-only access |
| Contributor | Create and modify resources |
| Owner | Full administrative control |

---

## RBAC Components

```text
User
   ↓
Assigned Role
   ↓
Defined Scope
   ↓
Resource Access
```

# 17. Azure Resource Hierarchy

Azure organizes resources using a hierarchical structure.

---

# Hierarchy Structure

```text
Management Group
        ↓
Subscription
        ↓
Resource Group
        ↓
Resource
```

---

# 17.1 Management Group

Top-level governance container.

---

# 17.2 Subscription

Billing and resource isolation boundary.

---

# 17.3 Resource Group

Logical container for related resources.

---

# 17.4 Resource

Actual cloud service.

### Examples

- Virtual Machine
- Database
- Storage Account
