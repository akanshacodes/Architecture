# 1️⃣ WHAT IS A PATH IN LINUX?

A path tells Linux:
```text
Where a file or folder exists
```

Just like a home address tells:
```text
Where a person lives
```

Linux path tells:
```text
Where a file/folder is stored
```

---

# 🌐 Real-Life Example

```text
India → Uttar Pradesh → Ghaziabad → Flat Number
```

This is also a path.

Similarly in Linux:

```text
/var/www/html/index.html
```

means:
```text
index.html file exists inside html folder
```

---

# 3️⃣ WHAT IS AN ABSOLUTE PATH?

Absolute Path means:
```text
Complete path starting from Root Directory (/)
```

---

# 🌐 Simple Definition

```text
Absolute Path ALWAYS starts with /
```

---

# 📌 Example

```bash
cd /home/chintu/chinki
```

---

# 🌐 Meaning

```text
Go to:
Root (/)
   ↓
home
   ↓
chintu
   ↓
chinki
```

---

# 5️⃣ ABSOLUTE PATH REAL EXAMPLES

| Absolute Path | Meaning |
|---|---|
| `/home/ubuntu` | ubuntu folder inside home |
| `/var/log` | log folder inside var |
| `/etc/nginx/nginx.conf` | nginx config file |
| `/var/www/html` | Nginx website folder |

---

# 🎯 DevOps Example

```bash
cd /var/www/html
```

Used to access:
```text
Nginx Website Directory
```

---

# 6️⃣ WHAT IS A RELATIVE PATH?

Relative Path means:
```text
Path relative to current location
```

---

# 🌐 Simple Definition

```text
Relative Path DOES NOT start with /
```

---

# 📌 Example

```bash
cd ../munna
```

---

# 🌐 Meaning

```text
Go one folder back,
then enter munna folder
```

---

# 8️⃣ UNDERSTANDING `.` and `..`

VERY IMPORTANT LINUX CONCEPT.

---

# 🌐 `.` (Single Dot)

Means:
```text
Current Directory
```

---

# 📌 Example

```bash
./script.sh
```

Meaning:
```text
Run script from current folder
```

---

# 🌐 `..` (Double Dot)

Means:
```text
One Directory Back
```

---

# 📌 Example

```bash
cd ..
```

Moves:
```text
One folder back
```

---

# 🌐 Example Visualization

```text
/home/chintu/chinki
```

Running:

```bash
cd ..
```

moves to:

```text
/home/chintu
```

---

# 9️⃣ RELATIVE PATH REAL EXAMPLES

| Relative Path | Meaning |
|---|---|
| `../` | One folder back |
| `../../` | Two folders back |
| `./app` | app folder in current directory |
| `logs/error.log` | error.log inside logs folder |

---

# 🎯 Example

Suppose current location:

```text
/home/ubuntu/project
```

Command:

```bash
cd logs
```

Final location:

```text
/home/ubuntu/project/logs
```

---

# 🔟 ABSOLUTE vs RELATIVE PATH COMPARISON

| Feature | Absolute Path | Relative Path |
|---|---|---|
| Starts With | `/` | Current Location |
| Full Address | Yes | No |
| Depends on Current Directory | No | Yes |
| Portable | Less | More |
| Used in Automation | Less | More |

---

# 🌐 Example Comparison

| Type | Command |
|---|---|
| Absolute | `cd /home/ubuntu/project` |
| Relative | `cd ../project` |

# 1️⃣1️⃣ WHY RELATIVE PATH IS USED IN AUTOMATION

MOST IMPORTANT DEVOPS CONCEPT.

The PDF says:
```text
Automation me hmesha relative path use hota hai
```

---

# 🌐 Why?

Automation scripts should:
- Work on multiple systems
- Work dynamically
- Avoid hardcoded locations

---

# ❌ Bad Practice

```bash
cd /home/ubuntu/project
```

Hardcoded path.

---

# ✅ Better Practice

```bash
cd ../project
```

Flexible and reusable.

---

# 🎯 Real DevOps Understanding

CI/CD pipelines prefer:
```text
Relative Paths
```

because environments change frequently.

# 1️⃣3️⃣ IMPORTANT INTERVIEW QUESTIONS

---

# ❓ What is an Absolute Path?

### ✅ Answer

An absolute path is a complete path starting from the root directory (`/`).

---

# ❓ What is a Relative Path?

### ✅ Answer

A relative path is a path based on the current working directory.

---

# ❓ What does `..` mean in Linux?

### ✅ Answer

`..` represents one directory back.

---

# ❓ What does `.` mean in Linux?

### ✅ Answer

`.` represents the current directory.

---

# ❓ Why are Relative Paths preferred in Automation?

### ✅ Answer

Because they make scripts portable and reusable across environments.

# 🏆 FINAL UNDERSTANDING

```text
Absolute Path:
Full exact address from root (/)

Address relative to current location

Absolute paths are fixed.
Relative paths are dynamic.

DevOps automation mostly prefers relative paths
because they work better across multiple environments.
```
