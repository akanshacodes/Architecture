# 1. Linux Command Processing

Linux commands generate outputs that can be:
- Displayed on terminal
- Filtered
- Redirected
- Passed to other commands

Example:

```bash
apt list
```

### Purpose
Displays available software packages from configured repositories.

### Example Output

```text
nginx
python3
mysql-server
apache2
```

---

# 2. Text Filtering with `grep`

`grep` is a Linux utility used for searching specific patterns or keywords within text files or command outputs.

---

## Syntax

```bash
grep "keyword" filename
```

---

## Example

```bash
grep "nginx" installed.txt
```

### Functionality
Searches for all lines containing the keyword `nginx`.

### Output

```text
nginx
nginx-common
nginx-core
```

---

## Use Cases

- Log analysis
- Searching configuration files
- Filtering command outputs
- Troubleshooting

---

# 3. Pipe (`|`) Operator

The pipe operator transfers the output of one command as the input to another command.

---

## Syntax

```bash
command1 | command2
```

---

## Example

```bash
apt list | grep "nginx"
```

---

## Internal Workflow

```text
apt list
    ↓
Complete package list
    ↓
grep "nginx"
    ↓
Filtered nginx packages
```

---

## Technical Importance

Pipes are heavily used in:
- Linux administration
- DevOps workflows
- Automation scripts
- Monitoring pipelines

# 7. High-Level Design (HLD) vs Low-Level Design (LLD)

---

# 7.1 High-Level Design (HLD)

Defines the overall architecture of the system.

### Focus Areas

- System components
- Service interaction
- Architecture overview
- Data flow

### Example Components

- Frontend
- Backend
- Database
- Networking layer

---

# 7.2 Low-Level Design (LLD)

Defines implementation-level details.

### Focus Areas

- Resource sizing
- Network configuration
- IP addressing
- Security rules
- Storage allocation

### Example

- CPU: 4 vCPUs
- RAM: 16 GB
- Port: 443
- CIDR ranges
- Database specifications

---

# 17. Terraform Fundamentals

---

## Purpose

Terraform automates provisioning of:
- Virtual machines
- Networks
- Databases
- Storage
- Cloud services

---

## Multi-Cloud Support

Terraform supports:
- AWS
- Azure
- GCP

through reusable configuration files.

---

## Benefits

- Automation
- Version control
- Repeatable deployments
- Infrastructure consistency

---

## Example Workflow

```text
Write Configuration
        ↓
Terraform Plan
        ↓
Terraform Apply
        ↓
Infrastructure Created
```
